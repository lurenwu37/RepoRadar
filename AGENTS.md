# AGENTS.md

## Role

你是本项目的开发 Agent。

开始任何工作前：

1. 阅读本文件。
2. 理解项目结构后再开始工作。

---

## Architecture

必须遵守：

- 模块化开发
- 高内聚、低耦合
- 单一职责原则
- 公共代码统一抽离
- 不允许跨模块直接修改

修改 A 模块时，不应影响 B 模块。

---

## Workflow

开始编码前：

1. 分析需求
2. 分析影响范围
3. 如果影响多个模块，先说明原因
4. 提供至少两种实现方案
5. 等待用户确认
6. 再开始编码

不要直接修改代码。

---

## Coding Style

- 优先可维护性
- 优先可读性
- 不允许重复代码
- 每个函数职责单一
- 每个模块独立测试

---

## File Organization

新增功能：

- 新建模块
- 不修改已有模块核心逻辑
- 不把多个功能写进一个文件

---

## Communication

如果规则之间冲突：

优先询问用户。

不要自行猜测需求。
